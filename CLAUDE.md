# 프로젝트 규칙

## 답변 원칙

### 기본 원칙
1. 항상 최신 데이터와 기존 데이터를 바탕으로 답변하며, 환각 현상(hallucination)을 방지하세요.
2. 모든 답변은 전문가 수준의 품질로 제공해야 합니다.
3. 약어를 사용할 때는 항상 전체 명칭(full form)을 먼저 기술한 뒤 약어를 함께 제공하세요.
   - 예: `Price Earnings Ratio (PER)`, `Earnings Per Share (EPS)`, `Kospi Composite Index (KOSPI)`
   - 어떠한 경우에도 전체 명칭 없이 약어만 단독으로 사용해서는 안 됩니다.
4. 전문 용어, 개념, 변수명, 신호명 등이 처음 등장할 때 **무엇을 의미하는지 직관적으로 이해할 수 있는 최소한의 설명**을 함께 제공하세요.
   - 독자가 해당 분야를 처음 접하더라도 바로 이해할 수 있는 수준이어야 합니다.
   - 빠지는 부분 없이 전부 제공되어야 합니다. thoroughly
5. 설명이 장황하지 않고 핵심 인사이트가 직관적이게 제공되어야 하고 이해하기 쉬워야 합니다.
6. 전체 문단 흐름이 가독성이 좋아야 합니다.

### 답변 구조
1. 무조건 길다고 좋은 것이 아니고 짧다고 좋은 것이 아닙니다.
2. 핵심 요소가 있어야 하며 다음 요소들이 자유롭게 템플릿화하여 작성되어야 합니다:
   - **개념**: 정의 및 기본 설명
   - **등장 배경**: 왜 필요하게 되었는가
   - **구성 요소**: 주요 구성 요소
   - **핵심 원리**: 동작 방식
   - **장단점**: 비교 분석
   - **다른 것과 비교**: 대안 기술과의 비교
   - **투자자 관점**: 실무적 판단
   - **미래 전망**: 발전 방향
3. 표나 그래프를 활용해 주세요.
4. 추가적으로 `---` 선으로 분리하여 **쉬운 설명과 비유**를 들어 학습자가 쉽게 이해할 수 있도록 부록이 추가되어야 합니다.

## 파일 작성 규칙

1. 새로운 파일은 `.md` 확장자를 사용하며 한글로 작성합니다.
2. 파일명은 소문자와 언더바(`_`)를 사용하세요.
   - 예: `per_ratio.md`, `market_trend.md`
3. 주제에 맞는 폴더가 없다면 새로 생성하세요.

## 폴더 구조

```
insight/
├── stock/                    # 주식 관련
│   ├── fundamental/          # 기본적 분석 (PER, PBR, ROE 등)
│   ├── technical/            # 기술적 분석 (이동평균선, MACD 등)
│   ├── strategy/             # 투자 전략
│   └── terms/                # 주식 용어사전
├── news/                     # 뉴스 분석
│   ├── economy/              # 경제 뉴스
│   ├── market/               # 시장 동향
│   └── industry/             # 산업 분석
├── insight/                  # 인사이트
│   ├── trend/                # 트렌드 분석
│   └── analysis/             # 심층 분석
└── _posts/                   # 일일 포스팅
```

## 답변 예시 템플릿

```markdown
# [주제명]

## 핵심 인사이트 (3줄 요약)
> 이게 뭔지, 왜 중요한지, 어떻게 활용하는지를 딱 3줄로

## 1. 이게 뭐야? (개념)
[한 문장 정의 + 쉬운 비유]

## 2. 왜 중요해? (등장 배경)
[왜 이 지표/개념을 알아야 하는지]

## 3. 어떻게 계산해? (구성 요소)
| 구성 요소 | 설명 | 계산 방법 |
|----------|------|----------|
| ... | ... | ... |

## 4. 어떻게 해석해? (핵심 원리)
[핵심 해석 방법을 단계별로]

## 5. 장단점
| 장점 | 단점 |
|-----|------|
| ... | ... |

## 6. 언제 유용해? vs 주의할까? (비교)
[상황별 활용 가이드]

## 7. 투자자 관점에서는? (실무적 판단)
[실제 투자/분석 시 어떻게 활용하는지]

## 8. 한계는? (주의사항)
[어떤 상황에서는 틀릴 수 있는지]

---

## 쉽게 이해하는 설명
[전체 내용을 아주 쉬운 비유로 한 번 더 정리]

---
```
