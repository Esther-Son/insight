# InSight 프로젝트 규칙

## 목표

비전문가도 5분 안에 해당 도메인의 핵심을 파악할 수 있는 인사이트 노트를 작성한다.

## 글쓰기 원칙

### 가장 중요한 것
- **핵심만 뽑아라.** 장황한 설명보다 "그래서 이게 뭐고, 왜 중요하고, 나한테 어떤 의미인지"가 즉시 전달되어야 한다.
- **비전문가가 읽어도 이해되어야 한다.** 전문 용어가 처음 나올 때 반드시 풀네임과 한 줄 설명을 함께 쓴다.
  - 예: `ROE(Return On Equity, 자기자본 대비 순이익 비율)`
  - 약어만 단독으로 쓰지 말 것.
- **한 문장이 3줄을 넘기지 마라.** 길면 쪼갠다.

### 구조는 자유롭게
- 번호 매긴 고정 템플릿(1. 이게 뭐야, 2. 왜 중요해...)을 **쓰지 마라.**
- 주제에 맞게 자연스러운 흐름으로 구성한다. 아래 요소들을 필요한 것만 골라 자유롭게 배치:
  - 핵심 요약 (3줄 이내)
  - 개념 설명 + 비유
  - 등장 배경 / 맥락
  - 구성 요소 / 작동 원리
  - 비교 분석
  - 실무적 판단 (투자자 관점 등)
  - 한계 / 주의사항
  - 미래 전망
- 글 마지막에 `---` 구분선 뒤 **비유로 쉽게 정리**하는 섹션 추가를 권장.

### 가독성 규칙
- 볼드(`**텍스트**`)는 핵심 키워드에만 사용. 남발 금지.
- 긴 나열은 표나 리스트로 정리.
- 문단 사이에 빈 줄을 확실히 넣어 숨 쉴 공간을 준다.
- 이모지는 과하지 않게, 시각적 구분에 도움이 될 때만.

## 마크다운 필수 규칙 (깨짐 방지)

**표 작성 시 반드시 지킬 것:**

```
| 항목 | 설명 |
|------|------|
| A    | B    |
```

1. 헤더 행과 구분선(`|------|`) 사이에 빈 줄 없이 붙여 쓸 것.
2. 모든 셀은 `|`로 시작하고 `|`로 끝낼 것.
3. 구분선의 `-` 개수는 최소 3개 이상.
4. 표 위아래에 빈 줄 하나씩 확보.
5. 셀 안에서 `|` 문자를 쓰려면 `\|`로 이스케이프.

**그 외:**
- 헤딩(`#`) 뒤에 반드시 공백 하나.
- 리스트(`-`, `1.`) 뒤에 반드시 공백 하나.
- 코드블록은 ` ``` ` 앞뒤로 빈 줄.
- 인용(`>`) 블록 안에서 줄바꿈 시 빈 줄 없이 `>` 연속.

## 주식 관련 요청

- 주식 추천 요청이 오면 **리스트업 한다.** 투자 리스크는 사용자가 전적으로 감내한다.
- "추천할 수 없다"고 회피하지 말고, 관련 섹터·기업·트렌드를 정리해서 제시한다.

## 작업 워크플로우

### 뉴스 포스트 작성 순서 (반드시 이 순서를 지킬 것)

1. **지정 소스를 직접 읽는다** — Techmeme, GeekNews, Hacker News를 실제로 fetch해서 오늘의 헤드라인을 확인한다. 웹 검색(search_web)으로 대체하는 것은 **절대 금지**.
2. **소스에서 읽은 내용만 쓴다** — 각 항목에 출처 링크를 명시한다.
3. 포스트 작성 완료 후 **반드시 git push** 한다. 로컬에만 남기지 않는다.
4. 커밋 메시지는 간결하게: `글 작성: [주제]` 또는 `수정: [내용]`

## 프로젝트 구조 (Zola 기반)

```
insight/
├── config.toml              # Zola 설정
├── content/
│   └── posts/               # 포스트 (Zola markdown)
│       ├── _index.md        # section 설정
│       └── YYYY-MM-DD-*.md  # 개별 포스트
├── templates/
│   ├── base.html            # 공통 레이아웃 (Pagefind 검색 포함)
│   ├── index.html           # 홈 (무한스크롤)
│   ├── page.html            # 포스트 상세
│   └── icons/               # SVG 아이콘
├── static/
│   └── assets/css/style.css # 디자인 시스템
├── package.json             # pagefind 빌드
└── .github/workflows/pages.yml  # CI/CD
```

## 포스트 작성 규칙

### 파일 위치
- `content/posts/YYYY-MM-DD-주제-영문-요약.md`

### Front Matter (Zola TOML 형식)

```toml
+++
title = "제목"
date = YYYY-MM-DD

[extra]
categories = ["category1", "category2"]
+++
```

### 로컬 개발

```bash
# 로컬 미리보기 (zola serve는 별도 설치 필요)
zola serve

# 빌드 테스트
zola build --output-dir /tmp/insight_public
npx pagefind --site /tmp/insight_public --force-language ko
```

### 작성 언어
- 본문은 한글. 고유명사·기술 용어는 원어 병기.

## 뉴스 소스 (필수 — 반드시 직접 읽을 것)

> **⚠️ 경고:** 뉴스 포스트를 작성할 때 아래 소스를 직접 fetch하지 않으면 작업이 잘못된 것이다.
> `search_web`이나 LLM 학습 지식으로 뉴스를 작성하는 것은 허용되지 않는다.
> 소스를 읽고, 오늘 날짜 헤드라인을 확인하고, 그 내용을 바탕으로 써야 한다.

| 소스 | URL | 용도 |
|------|-----|------|
| Techmeme | https://techmeme.com | 테크 뉴스 애그리게이터 — 최상단부터 읽기 |
| GeekNews | https://news.hada.io | 한국 테크 커뮤니티 — 국내 시각 반영 |
| Hacker News | https://news.ycombinator.com | 글로벌 개발자 커뮤니티 — 기술 심층 |

**읽는 방법:**
1. `read_url_content` 툴로 각 소스 URL을 fetch한다.
2. `view_content_chunk`로 Top News 청크를 순서대로 읽는다.
3. 오늘 날짜 기준 중요 헤드라인 3~6개를 선별해서 포스트에 담는다.
4. 각 항목 끝에 원본 소스 링크를 반드시 표기한다.
