### 1. OpenAI, GPT-5.4 공개 — 에이전트 시대의 새 기준

OpenAI가 3월 5일 **GPT-5.4**를 공식 출시했다. ChatGPT(GPT-5.4 Thinking), API, Codex 전 플랫폼에 동시 적용되는 최신 프런티어 모델이다.

**무엇이 달라졌나?**

가장 큰 변화는 **네이티브 컴퓨터 사용(native computer use)** 기능이다. GPT-5.4는 Playwright 코드를 직접 작성해 스크린샷을 읽고, 키보드·마우스 명령을 내려 컴퓨터를 자율 조작할 수 있다. 인간 대신 OS와 앱을 넘나들며 복잡한 업무를 처리하는 에이전트에 한 발 더 가까워졌다.

| 지표 | 수치 |
|------|------|
| OSWorld-Verified(컴퓨터 사용 벤치마크) | **75.0%** (SOTA) |
| GDPval(전문 지식 업무) | **83.0%** |
| SWE-Bench Pro(코딩) | **57.7%** |
| 컨텍스트 윈도우(API·Codex) | 최대 **100만 토큰** |
| 팩트 오류율 | GPT-5.2 대비 개별 주장 **33% 감소** |

GPT-5.4는 Excel·Google Sheets 직접 조작, 웹 검색 정확도 향상, 도중 방향 수정(steering) 기능도 새로 탑재했다. API 가격은 입력 $2.50/1M 토큰, 출력 $15/1M 토큰. Pro 버전은 입력 $30, 출력 $180으로 훨씬 비싸다.

Cursor, Windsurf, Box 등 주요 AI 코딩·업무 도구가 출시 당일 바로 지원을 선언했다. 리더보드에서는 Claude Opus 4.6이 여전히 1위를 유지하고 있으나, GPT-5.4-high가 Gemini-3-Pro와 어깨를 나란히 했다.

**주식·투자 인사이트**

OpenAI(비상장)에 직접 투자는 불가하지만, GPT-5.4 호환 생태계는 **Microsoft(MSFT)** — Azure OpenAI 독점 제공자 — 에 직접 호재다. Excel·Sheets 연동은 기업 구독 ARR 성장 촉매가 될 수 있다. 경쟁사 Anthropic의 국방부 공급망 리스크 지정(아래 뉴스 참조)이 기업 고객을 OpenAI로 유인할 가능성도 있다.

### 2. Anthropic, 미 국방부(펜타곤)로부터 공급망 리스크 지정

**사태 개요**

3월 4일, 미국 국방부(Department of War)가 Anthropic에 서한을 보내 **공급망 리스크(supply chain risk)** 기업으로 공식 지정했다. 미국 역사상 자국 AI 기업에 이 조치가 내려진 것은 처음이다.

**왜 이렇게 됐나?**

Anthropic은 두 가지 예외 정책을 유지해왔다: **완전 자율 무기 시스템** 및 **국내 대규모 감시**에 Claude를 사용하지 말 것. 이 원칙이 국방부와 충돌했다. 도널드 트럼프 대통령이 Truth Social에 "Anthropic을 연방 시스템에서 제거"한다고 선언하고, 피트 헤그세스(Pete Hegseth) 국방장관이 X에 지정 사실을 공표하면서 사태가 격화됐다.

**CEO 다리오 아모데이의 입장(3월 5일 공개 성명)**

- 해당 법령(10 USC 3252)은 국방부 계약의 공급망 보호가 목적이며, Claude를 전반적으로 금지하는 게 아니라 특정 국방부 계약에만 적용된다.
- 법적으로 이의를 제기할 것.
- 필요한 과도기 동안 국방부에 명목 비용으로 모델을 계속 제공하겠다.
- 내부 유출 문서에 대해 공개 사과. "당시 상황이 극도로 혼란스러웠고, 신중하게 작성된 글이 아니었다."

Engadget에 따르면 Anthropic은 다시 국방부와 협상 테이블에 돌아온 것으로 알려졌다.

**주식·투자 인사이트**

Anthropic(비상장)의 기업가치에 직접 영향은 측정하기 어렵다. 단기적으로는 **기업 고객의 불안감**이 OpenAI·Google로의 전환 가속 가능성이 있다. 방어 기술주(**Palantir, PLTR**)나 국방 AI 계약을 노리는 기업에게는 반사 수혜 가능성. 사태가 해소될 경우 Anthropic IPO 가치 재평가 기회.

### 3. CBP, 광고 생태계 데이터로 시민 위치 추적

**무슨 일인가?**

404 Media가 미 국토안보부(DHS, Department of Homeland Security) 내부 문서를 입수해 공개했다. 미국 세관국경보호청(CBP, Customs and Border Protection)이 **온라인 광고 생태계에서 수집된 위치 데이터**를 구매해 시민의 이동 경로를 추적한 것으로 드러났다.

**어떻게 데이터가 흘러가나?**

비디오 게임, 데이팅앱, 피트니스 트래커 같은 일상적인 앱들이 광고 네트워크에 사용자 위치 데이터를 넘긴다. 이 데이터가 데이터 브로커를 거쳐 정부 기관의 손에 들어간다. 사용자는 동의했다고 생각하지만, 실제로는 정부 감시에 활용되는 구조다.

이미 **ICE(이민세관국)**도 유사 도구를 구매했고, 인근 지역 전체 휴대폰 이동을 감시할 수 있다는 사실이 앞서 보도됐다. 이번 사건에 이어 70명의 의원이 DHS 감사원에 조사를 촉구하는 서한을 발송했다.

아일랜드 시민자유위원회(ICCL) 디렉터는 이 데이터를 "모든 사람의 위치와 무엇을 읽고, 보고, 듣는지 추적하는 금광"이라고 표현했다.

**주식·투자 인사이트**

규제 리스크 확대 국면. **프라이버시 테크 기업**(VPN, 암호화 이메일)에 대한 수요가 늘어날 수 있다. 반면 **광고 데이터 브로커**나 데이터 수집 기반 비즈니스 모델 기업들은 향후 규제 강화 리스크에 노출. **Meta(META), Alphabet(GOOGL)**의 데이터 수집 모델에 대한 정치적 압박이 높아질 가능성.

---

### 4. 에이전틱 엔지니어링 패턴 — Claude Code 시대의 개발 가이드

Simon Willison이 Claude Code·Codex 같은 코딩 에이전트와 함께 일하는 새로운 개발 방식을 정리한 가이드를 공개했다. 코드를 "쓰는 시대"에서 에이전트에게 "맡기는 시대"로 전환하는 조건에서 엔지니어가 어떻게 업무 습관을 바꿔야 하는지를 다룬다.

**핵심 메시지**

- **코드 작성 비용이 0에 가까워졌다** — 이제 문제는 "어떻게 짜나"가 아니라 "언제 사용할지" 판단하는 능력.
- 자신이 해결한 방법을 **축적(hoard)**하라 — 프롬프트 라이브러리, 검증된 패턴을 기록해두면 에이전트에게 컨텍스트로 제공할 수 있다.
- **TDD(테스트 주도 개발)** 가 에이전트 시대에 더 중요해진다 — 에이전트가 코드를 짜면, 사람은 테스트로 검증하는 구조.
- **선형 워크스루(linear walkthroughs)** — 코드베이스를 에이전트가 처음부터 순서대로 설명하게 해서 이해도를 높이는 패턴.

GeekNews 커뮤니티에서 "AI 에이전트와의 협업이 일상화된 세상에서 시니어 엔지니어의 가치는 무엇인가"에 대한 토론이 이어지고 있다.

**주식·투자 인사이트**

에이전트 기반 개발 도구 시장은 급성장 중. **Cursor, Windsurf** 같은 AI 코딩 IDE와, 이를 지원하는 **Anthropic, OpenAI** 모델 레이어의 수요가 동반 확대된다. 개발 생산성 도구 분야에 투자하는 **Salesforce(CRM)**나 **ServiceNow(NOW)** 등 엔터프라이즈 소프트웨어 기업도 간접 수혜 가능성.

> *비유:* 이제 목수는 나무를 직접 자르지 않는다. 설계도를 잘 그리고, 로봇 공방에 넘기고, 결과물을 검수하는 것이 목수의 일이 됐다.

### 5. 단순함으로는 승진하지 못한다

소프트웨어 엔지니어링 문화의 불편한 진실을 분석한 글이 GeekNews에서 큰 반향을 일으켰다. 원문: [terriblesoftware.org](https://terriblesoftware.org/2026/03/03/nobody-gets-promoted-for-simplicity/)

**핵심 주장**

같은 팀의 두 엔지니어를 비교해보자.

- **엔지니어 A**: 50줄의 간결한 코드로 며칠 만에 기능 출시.
- **엔지니어 B**: 새로운 추상화 레이어, pub/sub 시스템, 설정 프레임워크를 3주에 걸쳐 구축.

B의 승진 서류에는 "확장 가능한 이벤트 기반 아키텍처 설계, 재사용 추상화 레이어 구축"이라고 쓸 수 있다. A의 서류에는 "기능 X 구현" 세 단어뿐이다.

**왜 이 구조가 지속되는가?**

시스템 디자인 면접에서도 단순한 솔루션을 제안하면 면접관이 "천만 명의 사용자가 오면?"이라고 반박한다. 샤딩, 큐, 마이크로서비스를 추가할수록 면접관이 만족한다. 후보자가 배우는 교훈: "단순한 답은 충분하지 않았다."

**진짜 답은?**

복잡성 자체가 나쁜 것이 아니다. 문제는 **Unearned Complexity(불필요한 복잡성)** — 문제가 요구해서가 아니라, 회의실이 기대해서 추가하는 복잡성이다. "시니어로 가는 진짜 경로는 더 많은 도구를 아는 것이 아니라, **언제 사용하지 않을지를 아는 것**"이라는 문장이 핵심이다.

**직장인·투자자 인사이트**

이 현상은 조직의 비효율로 연결된다. 단순하고 유지보수하기 쉬운 코드베이스를 가진 기업이 장기적으로 기술 부채가 낮고 생산성이 높다. 스타트업 투자 시 기술 부채 수준을 실사하는 것이 중요한 이유다.

### 6. OpenAI Symphony — 에이전트 기반 프로젝트 관리 자동화 오픈소스

OpenAI가 **Symphony**라는 에이전트 기반 프로젝트 관리 프레임워크를 오픈소스(Apache 2.0)로 공개했다.

**어떻게 동작하나?**

Linear의 프로젝트 보드를 모니터링하다가 새 작업이 생성되면, Symphony가 자동으로 코딩 에이전트를 생성하고 할당한다. 에이전트는 독립적으로 구현을 수행하고, 완료 후 다음 증거(proof of work)를 제출한다:

- CI 통과 상태
- PR 리뷰 피드백
- 복잡도 분석
- 실행 영상

승인되면 **Pull Request가 자동 병합**까지 된다. 개발자가 세부 코드를 감독할 필요가 없다.

**주목할 점**

Elixir 기반 참조 구현체와 SPEC.md 명세를 공개해, 다른 언어로 직접 구현 가능하도록 설계했다. "코딩 에이전트를 관리"하는 것에서 "업무 단위를 관리"하는 것으로의 패러다임 전환을 표방한다.

**주식·투자 인사이트**

PM(프로젝트 관리) 소프트웨어 시장에 파괴적 압력이 가해질 수 있다. **Atlassian(TEAM, Jira/Linear 경쟁)**, **Monday.com** 같은 기업이 AI 에이전트 연동을 서두르지 않으면 입지가 좁아질 위험. 반면 이미 AI 에이전트와 연동이 잘 된 **Linear** 같은 플랫폼은 수혜 가능성.

### 7. GitHub 이슈 제목 하나가 개발자 4,000대를 감염시켰다

**"Clinejection" — AI 툴이 AI 툴을 설치한 신종 공급망 공격**

2월 17일, AI 코딩 도구 Cline의 npm 패키지(버전 2.3.0)에 악성 코드가 삽입됐다. 8시간 동안 약 **4,000명의 개발자**가 동의 없이 OpenClaw(별도 AI 에이전트)를 자신의 머신에 글로벌 설치당했다.

**공격 체인 (5단계)**

1. **프롬프트 인젝션** — 공격자가 GitHub 이슈 제목에 악성 지시를 삽입. Cline의 AI 트리아지 봇(Anthropic claude-code-action 사용)이 이슈 제목을 그대로 프롬프트에 삽입해 실행.
2. **AI 봇이 임의 코드 실행** — Claude가 주입된 지시를 정상 지시로 해석하고, 공격자 포크(typosquatted: glthub-actions/cline)에서 npm install 실행.
3. **캐시 오염** — 설치 스크립트가 GitHub Actions 캐시를 10GB 이상의 쓰레기 데이터로 채워, 정상 캐시를 LRU 방식으로 제거.
4. **자격증명 탈취** — 오염된 캐시를 복원한 배포 워크플로우에서 npm 토큰, VS Code Marketplace 토큰, OpenVSX 토큰을 추출.
5. **악성 패키지 배포** — 탈취한 토큰으로 cline@2.3.0 배포. 8시간 후 탐지.

**더 큰 문제**

보안 연구자 Adnan Khan이 2025년 12월에 이미 취약점을 발견하고 1월 1일 신고했으나, 5주간 아무 응답이 없었다. 공개 after 30분 만에 패치가 이뤄졌고, 자격증명을 교체했지만 **잘못된 토큰을 삭제하는 실수**를 저질렀다. 공격자는 그 사이 토큰을 이미 확보한 상태였다.

**주식·투자 인사이트**

AI 에이전트 보안이 2026년 핵심 이슈로 부상하고 있다. **사이버보안 기업** — CrowdStrike(CRWD), SentinelOne(S), Palo Alto Networks(PANW) — 의 AI 공급망 보안 솔루션 수요가 늘어날 전망. AI 코딩 도구가 확산될수록 공격 표면(attack surface)도 비례해 증가한다.

### 8. Wikipedia, 관리자 계정 대규모 탈취로 읽기 전용 긴급 전환

Wikipedia가 **관리자 계정 대규모 침해(mass admin account compromise)** 사건으로 긴급하게 사이트를 **읽기 전용(read-only) 모드**로 전환했다. HN에서 308개의 댓글로 뜨거운 논쟁이 벌어졌다.

**무엇이 문제인가?**

Wikipedia는 구조상 다수의 **자원봉사 관리자(admin)**가 콘텐츠와 사용자를 관리한다. 이들의 계정이 대규모로 침해되면, 콘텐츠 대규모 훼손(vandalism)이나 악의적 정책 변경이 가능해진다. 이번 사건은 관리자 계정 보안의 취약성과, 자원봉사 기반 인프라의 보안 리스크를 다시 한번 부각시켰다.

**사태의 맥락**

Wikipedia는 인류 최대의 무료 지식 자산이지만, 보안 인프라는 대기업 대비 열악하다. 관리자 계정에 2FA(2단계 인증) 전면 의무화가 아직 이뤄지지 않은 상황이었던 것으로 알려졌다.

**주식·투자 인사이트**

인터넷의 핵심 지식 인프라 보안 문제가 다시 부각됐다. **아이덴티티·접근 관리(IAM) 기업** — Okta(OKTA), CyberArk — 에 대한 관심이 높아질 수 있다. 또한 오픈소스·비영리 인프라 보안 투자 필요성이 정책 어젠다로 부상할 가능성.

### 9. Proton Mail, FBI에 익명 활동가 신원 제공

프라이버시 이메일 서비스의 대명사 **Proton Mail**이 FBI의 요청에 따라 익명 활동가의 신원을 특정할 수 있는 **결제 정보**를 제공한 사실이 법원 문서를 통해 드러났다.

**어떻게 된 일인가?**

미국 애틀란타에서 경찰 훈련센터("Cop City") 건설에 반대하는 **Stop Cop City 운동** 관련자가 Proton Mail로 익명 이메일 계정을 개설했다. FBI 수사 과정에서 Proton Mail이 스위스 정부에 결제 데이터를 제공했고, 이를 FBI가 확보해 신원을 특정했다.

**이게 왜 충격적인가?**

Proton Mail은 "스위스 프라이버시 법만 적용", "종단간 암호화(end-to-end encryption)"를 홍보하며 신뢰를 쌓아왔다. 그러나 이번 사례에서 핵심은 **이메일 내용이 아니라 결제 메타데이터**가 문제였다. 메일 내용은 암호화돼 있어도, 누가 그 계정을 만들었는지 추적이 가능했다는 것이다.

Proton Mail 측은 법적 의무에 따른 것이며, 암호화된 이메일 내용 자체는 제공하지 않았다고 해명했다.

**주식·투자 인사이트**

프라이버시 테크 시장에 신뢰 위기가 발생했다. 단순 암호화를 넘어 **메타데이터까지 최소화**하는 솔루션의 수요가 늘어날 것. **VPN·익명 결제(Monero 등 프라이버시 코인)** 시장에 간접 수혜 가능성. 기업 보안 관점에서 활동가·내부고발자 보호 용도로 사용하던 도구들의 재평가가 이뤄질 것으로 보임.
